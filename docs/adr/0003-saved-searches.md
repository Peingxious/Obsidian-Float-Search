# ADR-0003: 预设搜索（Saved Searches）

- 状态：Proposed
- 日期：2026-07-07

## Context

用户希望"加一个标签或指定内容，点一下就在边上搜出来"。标签本质属于属性类，因此主体是**属性类查询**，但也支持标签与任意搜索内容。需要可储存、可点击触发。

关键约束：预设查询使用 Obsidian 语法（`tag:#项目`、`[status:: 进行中]`），而 CMDK 极速框的自有引擎**不识别**这些运算符。因此预设不能由 CMDK 小框执行。

## Decision

数据结构：设置新增 `savedSearches: { name: string; query: string }[]`。

**创建（多入口，主在设置）**
- 主要在设置页：名称 + 查询 两个输入框 + "添加"按钮。
- 附加：在 CMDK / 主浮窗里提供"存为预设"快捷入口（捕获当前查询，填名即存）。

**访问（D 方案）**
- CMDK 浮窗顶部列出所有预设，作为**启动器**。
- 每条预设**自动生成一条 Obsidian 命令**（用户日常少用，留作后期筛选/绑定快捷键）。

**触发行为（见 ADR 结论 A）**
- 点预设 / 跑预设命令 → 打开**主浮窗（原生搜索视图）**，预填该查询并立即执行。CMDK 仅作入口，不出结果。
- 若已在主浮窗中再点预设，则把当前查询替换为预设并刷新。

## Consequences

- 预设统一走原生引擎，语法正确。
- 命令数量随预设增长（每个预设一条命令），Obsidian 命令面板会变长；可用命令命名前缀（如 `Float Search: 预设 - <名称>`）归类。
- 胶囊 UI 也可复用到预设的展示（可选，非必须）。
